Confix Architecture

6-axis parser architecture: JSON / YAML / CBOR

Axis Summary

  • • Axis 1: Json/Yaml/Cbor tokenizers → ConfixIndex
  • • Axis 2: Facet taxonomy (ConfixCol, ConfixIndexK)
  • • Axis 3: Span → native value decoders
  • • Axis 4: Scanner × Decoder → Cursor
  • • Axis 5: JsPath navigation over ConfixIndex
  • • Axis 6: CCEK lifecycle + ConfixFanout

ConfixIndex Facets

  • • Spans: Series<Twin<Int>> — open/close positions
  • • Tags: Series<IOMemento> — element types
  • • Depths: nesting depth per element
  • • DirectChildren: child indices
  • • TreeCursor: full tree as Cursor
  • • KeyToChild: name → child index map

Lifecycle States

  • • Created → Open: subscribe() window
  • • Open → Active: activate() launches coroutines
  • • Active → Draining: all subscribers complete
  • • Draining → Closed: supervisor.complete()
  • • ConfixFanout: one scope per syntax, concurrent
